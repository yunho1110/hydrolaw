## 프로젝트 소개

HydroLaw-AI는 복잡한 물환경 법령 및 지자체별 하수도 조례 정보를 영세 사업자와 소상공인이 쉽게 이해하고 준수할 수 있도록 돕는 공개 도구입니다. 사용자가 자연어로 자신의 업종·사업장 상황을 입력하면, RAG 파이프라인이 관련 법령을 검색하여 근거 조항과 함께 이해하기 쉬운 가이드라인을 생성합니다.

## 만든 이유

매번 개정되는 「물환경보전법」의 복잡한 배출허용기준(BOD, COD, SS 등)을 수동으로 찾아보고 해석하는 과정은 영세 사업자들에게 상당한 시간 부담이자 위반 리스크로 이어집니다. 이 프로젝트는 사용자가 간단한 자연어 질의만으로 최신 법령에 기반한 규제 준수 가이드라인을 즉시 확인할 수 있도록 하는 것을 목표로 시작되었습니다.

## 주요 기능

- **자연어 질의 분석**: 사용자가 입력한 상황(업종, 위치, 폐수 배출 형태 등)을 물환경 법령 및 배출허용기준 구조에 맞춰 분석
- **법적 근거 매칭**: 관련 법 조항과 맞춤형 대응 지침을 문서 형태로 제공
- **예시 기반 학습**: 레포지토리에 포함된 테스트 질의 예시로 질문 작성법과 챗봇 사용법을 빠르게 습득 가능
- **재사용 가능한 RAG 파이프라인**: 법령 문서를 매번 수동 검색해야 하는 반복 업무를 줄이는 기본 검색 흐름 제공

## 데이터 출처 및 최신성

- 기반 법령: 「물환경보전법」 및 관련 시행령·시행규칙 (국가법령정보센터 기준)
- 지식 베이스 갱신 시점: **2026-07-04** 기준 최신 개정본 반영 (배출허용기준은 시행규칙 별표 13, 최종
  개정 2021. 12. 10. / BOD·TOC·SS·총질소·총인 수치는 2020. 1. 1.부터 적용되는 현행 기준)
- 세부 출처·확인 방법·확인일자는 [`data/SOURCES.md`](data/SOURCES.md)에서 항목별로 확인할 수 있습니다.
- 지자체 조례는 현재 일부 지역만 우선 반영되어 있으며, 전국 확대는 로드맵 참조

> 법령은 수시로 개정되므로, 결과 확인 시 상단에 표기된 데이터 기준일을 반드시 확인하시기 바랍니다.

## 아키텍처 개요

```
사용자 자연어 입력
      │
      ▼
[질의 파서/전처리]
      │
      ▼
[검색(Retrieval)] ── 법령·조례 벡터 인덱스
      │
      ▼
[생성(Generation)] ── LLM 기반 응답 생성
      │
      ▼
근거 조항 + 대응 지침 (문서 출력)
```

### 하이브리드 구조: 숫자는 JSON, 맥락은 RAG

이 프로젝트의 핵심 설계 원칙은 "정확해야 하는 숫자"와 "맥락 설명"을 서로 다른 저장소로 분리하는
것입니다.

- **정확 조회가 필요한 수치**(BOD·TOC·SS·총질소·총인의 지역별·규모별 배출허용기준)는 오직
  `data/emission_standards.json`에만 존재하며, 파이프라인은 이 파일을 코드로 직접 조회해 값을
  가져옵니다. LLM이 검색된 텍스트 조각에서 숫자를 "추측"하도록 두지 않기 위한 장치입니다.
- **맥락·개념 설명**(법 조항의 취지, 지역구분·규모구분의 정의, 업종 분류, 행정처분 절차 등)은
  `data/laws/*.md`를 벡터 인덱스로 만들어 RAG로 검색합니다. 이 마크다운 문서에는 의도적으로
  배출허용기준 수치 표를 넣지 않으며, 수치가 필요한 자리는 "구체 수치는 배출허용기준 데이터(별표
  13) 조회 결과를 따름"이라는 문구로 대체되어 있습니다.

이렇게 분리해 두면 LLM이 검색 청크에 섞여 있는 숫자를 잘못 인용해 환각(hallucination)을 일으키는
경로 자체를 차단할 수 있습니다. 임베딩 모델·벡터스토어·검색 파라미터 등 RAG 파이프라인의 구체
구성은 `config.yaml`을 참고하세요.

## 설치 방법

```bash
git clone https://github.com/yunho1110/hydrolaw-ai.git
cd hydrolaw-ai
pip install -r requirements.txt
```

## 사용 방법

1. 레포지토리에 포함된 예시 파일(`examples/`) 및 `data/` 구조(`data/emission_standards.json`,
   `data/laws/*.md`)를 확인합니다.
2. 자신의 업종, 사업장 위치, 폐수 배출 상황 등 필요한 내용을 자연어로 입력합니다.

   ```bash
   python run.py --query "세차장을 운영 중인데 BOD 기준이 궁금합니다"
   ```
3. 실행 후 최신 물환경보전법 데이터베이스를 바탕으로 생성된 결과 및 법적 출처를 확인합니다.
4. 분석하고자 하는 타겟 지역(지자체)에 따라 `config.yaml`의 설정이나 조례 출력 형식을 수정합니다.
